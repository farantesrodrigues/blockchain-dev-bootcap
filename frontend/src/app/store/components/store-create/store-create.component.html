Create a new Store (a decentralized franchise of Company) by registering it's employee 0 - the owner - give him a name

A Store is a contract that holds a mapping of the different users involved in the store business :

The owner

The manager

The employee

The client

The permission set is decreasing with the list. The owner can register managers (and all others). A manager can register employees (and clients). An employee can register clients.

To interact with the contract, a few methods are available, such as registering new users, according to your set of permissions. To do that, once created, visit the store front (by front we mean the frontend side of the contract, in fact this is more like a back office)

The permissions are structured in an Enum called roles. Modifiers check before any user is registered that the msg.sender has the adequate level
